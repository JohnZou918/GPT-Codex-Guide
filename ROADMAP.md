# 项目路线图

版本顺序代表当前计划；若工具界面、安全要求或读者反馈变化，维护者会调整计划并在 [更新日志](CHANGELOG.md) 中说明。

| 版本 | 主题 | 计划成果 |
|---|---|---|
| V1.0 | 项目框架 | 目录、首页、章节大纲、许可证与维护规范 |
| V2.0 | Windows | 安装步骤、验证方法和常见问题 |
| V3.0 | macOS | 安装步骤、芯片差异、权限和验证方法 |
| V4.0 | Git | 版本管理与本地操作流程 |
| V5.0 | GitHub | 账号安全、仓库、同步和协作 |
| V6.0 | VS Code | 界面、文件、Markdown、终端和 Git |
| V7.0 | Codex | 任务描述、权限边界、差异检查与验收 |
| V8.0 | MCP | 模型上下文协议的用途、风险与连接 |
| V9.0 | AI Agent | 智能体的任务循环、监督与安全边界 |
| V10.0 | 企业开发 | 团队权限、审查、测试、发布与治理 |

## 共同完成标准

- 中文优先，专业术语首次出现时有注释。
- 回答是什么、为什么存在、有什么用途、怎样操作。
- 每个操作步骤解释为什么这样做。
- 说明正确结果、常见错误和低风险检查方法。
- Windows 与 macOS 差异有明确标记。
- 内部链接和 Markdown 格式通过检查。
