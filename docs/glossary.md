# 术语表

本术语表面向完全没有计算机基础的读者。解释以“它是什么、用来做什么”为重点，不要求读者先理解其他专业概念。

| 术语 | 中文解释 |
|---|---|
| AI | 人工智能。让计算机完成理解文字、识别内容、生成答案或协助决策等任务的一类技术。 |
| ChatGPT | OpenAI 提供的对话式 AI 产品。你可以用自然语言与它讨论问题、学习概念、整理需求和生成内容。 |
| Codex | OpenAI 的开发执行助手。它可以在获得允许的项目范围内读取文件、修改内容、运行命令并检查结果。 |
| Git | 一种版本管理工具。它记录文件在不同时间的变化，帮助你查看修改、保存阶段成果或回到较早版本。 |
| GitHub | 用于在线保存 Git 仓库和开展协作的网站。它不是 Git 本身，而是使用 Git 数据的在线服务。 |
| Repository（仓库） | 一个项目的专用文件夹及其版本记录，常简称 Repo。它通常包含文档、代码、图片和项目配置。 |
| Clone（克隆） | 把远程仓库完整复制到自己的电脑，并保留它与远程地址的关系。 |
| Commit（提交） | 给当前一组文件变化做一次有说明的版本记录，像为项目拍下一张可追踪的快照。 |
| Push（推送） | 把电脑上的新提交发送到 GitHub 等远程仓库。仅保存文件不等于已经推送。 |
| Pull（拉取） | 从远程仓库取得最新提交，并把它们合并到当前电脑的项目中。 |
| Branch（分支） | 从某个版本延伸出的独立工作线。它让你在不直接影响主线的情况下进行修改。 |
| Merge（合并） | 把一个分支中的修改整合到另一个分支。若同一位置被不同方式修改，可能需要人工选择。 |
| Main（主分支） | 仓库通常使用的主要分支名称，保存当前被认可的项目版本。 |
| Remote（远程仓库） | 位于 GitHub 等在线服务上的仓库副本。Git 常把主要远程地址命名为 origin。 |
| VS Code | Visual Studio Code 的简称，一种用于查看、编辑和管理项目文件的编辑器。 |
| Editor（编辑器） | 用来打开和修改文本、代码或配置文件的软件。VS Code 就是一种编辑器。 |
| Terminal（终端） | 通过输入文字命令操作电脑的窗口。Windows 和 macOS 都有终端工具。 |
| Command（命令） | 在终端中输入的一行操作指令，例如查看版本或切换文件夹。 |
| CLI | Command-Line Interface 的缩写，中文为“命令行界面”。它通过文字命令而不是鼠标菜单操作软件。 |
| GUI | Graphical User Interface 的缩写，中文为“图形用户界面”。它使用窗口、按钮和菜单供人操作。 |
| Path（路径） | 文件或文件夹在电脑中的位置说明，类似现实地址。Windows 与 macOS 的写法有所不同。 |
| Folder（文件夹） | 用来分类存放文件和其他文件夹的容器，也常称目录。 |
| Markdown | 一种使用简单符号排版文字的格式。文件通常以 .md 结尾，可表示标题、列表、链接和代码块。 |
| README | 项目首页说明文件，通常名为 README.md，用于介绍项目、使用方法和导航入口。 |
| License（许可证） | 说明别人可以如何使用、修改和分发项目内容的法律文件。 |
| Open Source（开源） | 公开项目源文件，并按照许可证允许他人查看、使用、修改或贡献的协作方式。 |
| Pull Request | 拉取请求，简称 PR。贡献者请求维护者审查并合并一组修改的 GitHub 协作功能。 |
| Issue | GitHub 中用于记录问题、建议、任务或讨论的条目。 |
| Fork（派生） | 在自己的 GitHub 账号下创建某个仓库的独立副本，常用于准备贡献。 |
| Conflict（冲突） | Git 无法自动判断两组修改应如何组合的情况，需要人选择正确内容。 |
| Homebrew | macOS 常用的软件包管理工具，可以用终端命令安装和更新许多开发工具。 |
| Package Manager（软件包管理器） | 帮助安装、更新和移除软件或程序依赖的工具。Homebrew 是一个例子。 |
| Node.js | 让 JavaScript 能在浏览器之外运行的软件环境，很多开发工具会依赖它。 |
| Runtime（运行环境） | 让某类程序能够执行所需的一组软件和条件。Node.js 是 JavaScript 的一种运行环境。 |
| Environment Variable（环境变量） | 操作系统保存的一类名称和值，程序可用它读取路径、配置或临时凭据。 |
| API | Application Programming Interface 的缩写，中文为“应用程序接口”。它规定不同软件如何按约定交换请求和结果。 |
| MCP | Model Context Protocol 的缩写，中文常译为“模型上下文协议”。它帮助 AI 应用以统一方式连接工具和数据来源。 |
| AI Agent（AI 智能体） | 能围绕目标规划步骤、调用工具、观察结果并继续行动的 AI 系统，仍需要人设定权限和监督。 |
| Token（令牌） | 根据语境可能指 AI 处理文字的基本单位，也可能指登录凭据。涉及账号时，令牌必须像密码一样保密。 |
| Authentication（身份验证） | 系统确认“你是谁”的过程，例如输入密码、使用验证码或登录授权。 |
| Permission（权限） | 规定某个用户或程序可以读取、修改或执行哪些内容的规则。 |
| Backup（备份） | 为重要数据创建额外副本，以便原文件丢失或损坏时恢复。 |
| Sync（同步） | 让不同设备或位置的数据保持一致。Git 的拉取和推送是项目同步的一部分。 |

## 使用与维护规则

- 正文首次出现术语时仍要就地解释，不能只让读者跳到术语表。
- 同一术语在全书使用一致译法。
- 一个词有多种含义时必须说明当前语境。
- 新增术语时按读者理解需要补充，不为追求数量堆砌词汇。
