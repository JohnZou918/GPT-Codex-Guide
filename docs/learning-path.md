# 学习路线

本路线面向零基础读者，建议每天学习 30～60 分钟、每周安排 4～5 天。时间可以调整；理解和实际验证比赶进度更重要。

## 开始前

- 确认自己使用 Windows 还是 macOS。
- 准备一个可以接收验证邮件的 GitHub 账号。
- 建立专门练习文件夹，不在重要工作文件上试验。
- 先阅读 [术语表](glossary.md)，不要求一次记住全部词语。

## 第一周：建立整体认识

学习：

- [第 1 章：整体认识](01-整体认识.md)
- [第 2 章：为什么安装这些软件](02-为什么安装这些软件.md)

目标：

- 说清 ChatGPT、Codex、Git、GitHub、VS Code 和操作系统各自负责什么。
- 区分网页服务、电脑软件、账号、本地文件和在线仓库。
- 根据自己的电脑建立安装清单。

## 第二周：完成电脑环境准备

Windows 用户学习：

- [第 3 章：Windows 安装](03-Windows安装.md)

macOS 用户学习：

- [第 4 章：Mac 安装](04-Mac安装.md)

目标：

- 从官方来源下载安装工具。
- 完成首次启动和版本验证。
- 记录自己的系统与安装结果。

## 第三周：认识 GitHub

学习：

- [第 5 章：GitHub 使用](05-GitHub使用.md)

目标：

- 理解账号、仓库、公开与私有的区别。
- 创建练习仓库并找到仓库地址。
- 认识 README、Issue 和 Pull Request 的基本用途。

## 第四周：学习 Git

学习：

- [第 6 章：Git 使用](06-Git使用.md)

目标：

- 理解工作区、提交和远程仓库。
- 完成查看状态、暂存、提交、拉取和推送。
- 知道保存文件、提交和推送是三件不同的事。

## 第五周：使用 VS Code

学习：

- [第 7 章：VS Code 使用](07-VSCode使用.md)

目标：

- 正确打开项目文件夹。
- 编辑并预览 Markdown。
- 使用集成终端和版本管理入口。

## 第六周：安全使用 Codex

学习：

- [第 8 章：Codex 使用](08-Codex使用.md)
- [第 9 章：ChatGPT 与 Codex 分工](09-ChatGPT与Codex分工.md)

目标：

- 写出包含目标、范围、禁区和验收标准的任务。
- 查看 Codex 修改内容并运行检查。
- 判断一项任务应该先讨论还是直接进入项目执行。

## 第七周：跨设备与固定流程

学习：

- [第 10 章：Windows 与 Mac 互联互通](10-Windows与Mac互联互通.md)
- [第 11 章：开发固定流程](11-开发固定流程.md)

目标：

- 在不同电脑上使用同一个远程仓库。
- 坚持工作前拉取、修改后检查、提交后推送。
- 建立可重复的小步开发流程。

## 第八周：排错与形成习惯

学习：

- [第 12 章：常见问题](12-常见问题.md)
- [第 13 章：每日操作清单](13-每日操作清单.md)

目标：

- 保留完整错误信息并按顺序排查。
- 使用每日清单完成安全开工和收尾。
- 独立完成一次从需求到 GitHub 复核的完整练习。

## 全部学完后的验收

- [ ] 我能解释整套工具的关系。
- [ ] 我能在自己的操作系统上验证基础工具。
- [ ] 我能创建、克隆、提交、拉取和推送练习仓库。
- [ ] 我能使用 VS Code 管理项目文件。
- [ ] 我能为 Codex 提供清晰边界并检查结果。
- [ ] 我能在不同设备间安全继续项目。
- [ ] 我能使用固定流程和清单减少错误。

完成本路线后，可按照 [项目路线图](../ROADMAP.md) 继续学习后续版本中的 MCP、AI Agent 和企业开发内容。
